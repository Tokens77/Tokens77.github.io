Trapdoor Commitment

方案目标:确保消息内容保密的同时，允许承诺方灵活处理消息的承诺方案。

符号说明:

κκκ:保密参数-security parameter

cppcppcpp:公开参数-public parameter

trtrtr:陷门参数-trapdoor

ϑ\varthetaϑ:随机数-random number

ccc:公开的承诺字符串-the commitment string

m:消息-message

C.setup(1κ1^{κ}1κ)

  • 输入:安全参数κ
  • 输出:公开参数cppcppcpp与陷门trtrtr

C.commit(cpp,m,ϑcpp,m,\varthetacpp,m,ϑ)

  • 输入:cpp,m,ϑcpp,m,\varthetacpp,m,ϑ
  • 输出:c

mmm为消息空间M\mathcal{M}M中的值，ccc为承诺空间C\mathcal{C}C中的值，这些都在公开参数cppcppcpp中

C.equivocal(cpp,c,(m,ϑ),m′,trcpp,c,(m,\vartheta),m',trcpp,c,(m,ϑ),m′,tr)

  • 输入:cpp,c,m,ϑ,m′,trcpp,c,m,\vartheta,m',trcpp,c,m,ϑ,m′,tr
  • 输出:ϑ′\vartheta'ϑ′

C.decommit(cpp,c,m′,ϑcpp,c,m',\varthetacpp,c,m′,ϑ)

在这个阶段，发送方会公布消息m′m'm′、随机数ϑ′\vartheta'ϑ′

接收方验证c=C.commit(cpp,m′,ϑ′)c=C.commit(cpp,m',\vartheta')c=C.commit(cpp,m′,ϑ′)，若成立则output(1)output(1)output(1)

优点:

1、可以有效抵御重放攻击、选择消息攻击、欺骗攻击以及后向欺骗攻击等

2、利用陷门trtrtr将于原本承诺值c关联的消息mmm，更新为与消息m′m'm′关联，使得能够通过陷门将承诺值ccc与不同的消息进行匹配，相当于给发送方提供了在特殊情况下修改消息的渠道。

总结:

  • 承诺方在拥有陷门trtrtr的条件下，可以选择是否修改承诺消息mmm

  • 而对于mmm是否已经被修改，这个问题不是接收方关心的，接收方只关心在decommit阶段，收到的消息m′m'm′、随机数ϑ′\vartheta'ϑ′是否与ccc匹配

RSA方案实现

C.setup(1κ1^{κ}1κ)

cpp取(n,e)，门限tr当作私钥

C.commit(cpp,m,ϑcpp,m,\varthetacpp,m,ϑ)

  • 发送方生成承诺c

    c=(m+ϑ)e  mod  nc=(m+\vartheta)^e\;mod\;n c=(m+ϑ)emodn

C.equivocal(cpp,c,(m,ϑ),m′,trcpp,c,(m,\vartheta),m',trcpp,c,(m,ϑ),m′,tr)

  • 目标:生成新的随机数ϑ′\vartheta'ϑ′，使得新承诺c′c'c′对新消息m′m'm′是有效的

  • 发送方先解密承诺值ccc有:m+ϑ=ctr  mod  nm+\vartheta=c^{tr}\;mod\;nm+ϑ=ctrmodn

    再生成新随机数ϑ′\vartheta'ϑ′:ϑ′=(ctr−m′)  mod  n\vartheta'=(c^{tr}-m')\;mod\;nϑ′=(ctr−m′)modn

C.decommit(cpp,c,m′,ϑ′cpp,c,m',\vartheta'cpp,c,m′,ϑ′)

  • 接收方计算c′c'c′

    c′=(m′+ϑ′)e  mod  nc'=(m'+\vartheta')^e\;mod\;n c′=(m′+ϑ′)emodn

    比较c=c′c=c'c=c′ 是否成立即可验证承诺。
